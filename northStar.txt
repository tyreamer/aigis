Rekall End State — The Git for AI Decisions (2028 North Star)
Rekall becomes the default coordination and governance infrastructure for agentic execution and governed workflows.
Core Reality
Every project maintains one or more local, git-portable Rekall Vaults — tamper-evident once shared/anchored, cryptographically hash-chained execution records that live alongside code. Agents discover and write to these vaults by MCP convention. Humans govern through simple CLI commands. No server is required for daily work.
How the system works at scale

AI agents coordinate by default: every decision, wait, policy check, and outcome is appended to the canonical verifiable execution record with signatures and hash chaining.
Humans retain full control via rekall decide, rekall status, and rekall verify.
Projects remain fully local-first and portable via rekall bundle.

When an organization needs enterprise oversight, any number of vaults federate upward into the Enterprise Rekall Hub (hosted control plane). The Hub provides:

Centralized policy enforcement
Verifiable provenance + best-effort replay of past executions
Tamper-evident audit trails for compliance and regulators

The result is infrastructure, not another tool: agents treat Rekall as the default substrate the same way they treat Git today. Enterprises mandate it the same way they mandate audit logging and SOC 2 controls. Every AI execution leaves a canonical, replayable, cryptographically proven record that is easier to reason about than Git history.
